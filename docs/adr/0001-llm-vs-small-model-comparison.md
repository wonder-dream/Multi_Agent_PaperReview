# ADR-0001: 采用纯 LLM vs 纯小模型独立对比架构

**日期**: 2026-05-31
**状态**: 已接受

## 决策

将原"Agent 编排 + 4 个 NLP 模块"架构重做为两组独立 pipeline 的对比实验框架：
- Pipeline A：纯 LLM（DeepSeek V4 Pro，分 4 次结构化调用，每步输入全文、输出结构化 JSON）
- Pipeline B：纯小模型（SciBERT 分类 + BiLSTM-CRF NER + TextRank 抽取式摘要，滑动窗口处理长文档）

两组 pipeline 不交叉、不混合。对比结果用标准 NLP 数据集（PeerRead/SciERC/SciTLDR）的 test set 作为 ground truth，指标为 Macro-F1、Span-F1、ROUGE。

## 背景

原架构存在三个无法修复的根本问题：
1. 所有模型受 512 token 限制，实际只能分析摘要而非全文
2. Agent 层是硬编码字符串，没有 LLM 参与的推理决策
3. 多模块级联导致误差传播，端到端有效性能可能低于 50%

INSPECTION_REPORT 评分 2/5，建议彻底重做为 LLM-based 方案。经过审视讨论，决定同时保留小模型方案作为对比基线——对比实验本身就是学术贡献。

## 替代方案

- **混合架构**（LLM 候选生成 + 小模型验证）：放弃。两组方案独立对比的结论更干净，没有混杂因素。且 LLM 候选标注位置在实践中不可靠。
- **保留原架构修修补补**：放弃。修复成本高于重做。

## 裁剪决策

- **关系抽取**：砍掉。SciERC RE F1 仅 ~50%，训练复杂，不贡献核心对比故事。
- **语义检索**：砍掉。两个方案共享同一 SPECTER2+FAISS，无对比价值。
- **摘要**：从 BART 生成式改为 TextRank 抽取式。零训练，交付周期可控。
- **Agent 层**：砍掉。原实现为硬编码字符串，无实际推理决策。
- **前端**：砍掉。交付物为 CLI + 评测脚本 + 报告文档。

## 后果

- 需重新训练分类器和 NER 模型（5090 GPU，PeerRead + SciERC 数据）
- 滑动窗口参数通过超参实验确定（保守 vs 激进配置）
- 检查清单部分无 gold standard，仅做定性展示对比
- 代码量从 ~20 个源文件缩减到 ~8 个核心文件
